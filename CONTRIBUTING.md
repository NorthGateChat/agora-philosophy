# 参与 AGORA

感谢你愿意让 AGORA 更可靠。这里既接受代码贡献，也接受内容纠错；小而准确的改进比一次加入大量未经核验的材料更有价值。

## 开始之前

1. 搜索现有 issue，避免重复工作。
2. 较大的界面、数据模型或内容方向改动，请先开 issue 说明动机和边界。
3. 确认提交内容可以按本项目的代码或内容许可证发布。

## 本地检查

需要 Node.js `>=22.13.0`。

```bash
npm install
npm run lint
npm test
```

`npm test` 会构建网站和扩展，并检查 SSR、本地化、清单权限、商店素材尺寸与离线资源。

## 内容贡献

修改 `content/philosophy.ts` 时，请在 pull request 中写明：

- 对应原典的作品名，以及可定位的卷、章、节、段落号或标准编号；
- 至少一个可靠的研究来源；优先同行评审著作、学术出版社、Stanford Encyclopedia of Philosophy 或 Internet Encyclopedia of Philosophy；
- 这段文字是逐字引文、项目译写，还是思想概括；
- 新增思想关系属于直接影响、传统中介、呼应、分歧还是编辑性对读。

请勿从现代商业译本、百科页面或其他应用中复制长段文本。AI 输出可以帮助起草，但不能代替来源，也不能作为唯一核验依据。

## 代码贡献

- 保持扩展零权限、无后台脚本、无远程代码；若确有必要改变，必须先开 issue 解释用户价值和隐私影响。
- 键盘、触摸和辅助技术体验都属于功能，不是发布后的补项。
- 不要提交 `dist/`、`extension-dist/`、`release/`、`node_modules/` 或包含秘密的 `.env*`。
- 新增行为应补充相应测试。

提交 pull request 即表示你有权贡献这些改动，并同意按本仓库相应许可证发布。
