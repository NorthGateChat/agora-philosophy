# AGORA 发布与自媒体传播 Playbook

产品全名：**AGORA｜哲学新标签页**

产品简称：**AGORA**

主 Slogan：**一页，一种思想。**
传播主张：**不是金句，是思想的来路与去处。**

## 1. 品牌与定位

AGORA 来自古希腊的公共广场：人们在那里相遇、讨论和辩论。产品不要求用户专门腾出一小时上课，而是在每次打开新标签页时，让一个观点成为继续阅读思想脉络的入口。

一句话定位：

> 一款把新标签页变成哲学入口的 Chrome 扩展：从一句命题出发，看见它的双语文本、解释、思想上下游与相似观点。

推荐品牌层级：

- 商店与首次出现：`AGORA｜哲学新标签页`；
- 产品界面与社交传播：`AGORA`；
- 英文商店名称固定使用 `AGORA — Philosophy New Tab`，所有语言均使用大写 `AGORA`，不另设译名；
- 主 Slogan 固定使用“一页，一种思想。”，不要在同一张物料上并列多个口号。

可用于后续 A/B 测试的备选句，而非首发并列口号：

1. 每一次新页，都是思想的入口。
2. 把新标签页，留给一个值得想的问题。
3. 在浏览之间，给思想一点空隙。

### 表达边界

可以说：

- 精选哲学命题；
- 从双语文本到解释，从前因到后续；
- 在日常浏览间隙学习哲学；
- 无账号、无追踪、无需权限。

不要说：

- “全网最全”“收录全部哲学家”“最权威”；
- “每条引文都是真理”或把二手转述伪装成逐字原文；
- “永久免费”“绝不增加任何功能”等无法长期保证的话；
- 用具体安装量、评分或媒体推荐作背书，除非当时可以公开验证。

## 2. Chrome Web Store 文案

### 商店名称

**AGORA｜哲学新标签页**

### 简短介绍

> 每次打开新标签页，遇见一句哲学命题，以及它的双语文本、思想脉络与相似观点。无账号、无追踪、无需权限。

### 详细介绍

> 一页，一种思想。
>
> AGORA 把 Chrome 新标签页变成一个安静的哲学入口。每次打开新页，你会遇见一则精选命题；点击它，可以继续阅读中英文译写、简要解释、思想的前因后续，以及其他哲学家的相似或分歧观点。
>
> 你可以：
>
> - 在简体中文、繁体中文和英文之间切换；
> - 选择中文句子是否同时显示英文译写；
> - 用键盘快捷键切换命题、打开详情或收藏；
> - 在不同视觉主题中选择适合自己的阅读氛围；
> - 收藏想再次回看的命题。
>
> AGORA 不要求注册账号，不包含广告或行为追踪，也不申请任何 Chrome 权限。语言、主题与收藏只保存在当前浏览器本地。
>
> 它不是一本哲学百科全书，也不试图用一句话代替原著。它想做的是：在日常浏览的间隙，给你一个继续追问的入口。

### 单一用途说明

> 在新标签页呈现精选哲学命题、双语文本、简要解释与思想关联，帮助用户在日常浏览间隙学习哲学。

### 首版更新说明

> AGORA 首个公开版本：支持简体中文、繁体中文和英文三种语言模式、双语命题、三种视觉主题、收藏与快捷键，并可从命题继续查看解释、上下游思想和相似观点。

## 3. 14 天发布节奏

在不接入第三方追踪的前提下，主要观察 Chrome Web Store 后台的聚合展示/安装数据、各平台公开互动、商店评论和内测者自愿反馈。

| 日期 | 目标 | 发布内容 | 行动与观察 |
|---|---|---|---|
| D-7 | 让受众理解问题 | “你的新标签页，每天被打开多少次？”静态预告 | 收集大家对默认新标签页的真实抱怨与期待 |
| D-6 | 建立视觉记忆 | 8 秒录屏：空白新页出现一则命题，断环标记轻动 | 测试“AGORA”的名字记忆度 |
| D-5 | 讲清差异 | 图文：不是金句，是思想的来路与去处 | 重点展示上下游与相似观点，而不只是首页 |
| D-4 | 建立内容可信度 | 一条命题从原典线索、双语译写、出处到关系图的制作过程 | 邀请哲学/翻译读者纠错，不用“权威”姿态 |
| D-3 | 回答安全顾虑 | 零权限、无账号、无追踪、本地收藏说明 | 发布隐私说明链接，招募重视隐私的内测者 |
| D-2 | 小范围实测 | 开放 15–30 个内测名额 | 用文末清单收集设备、版本和复现步骤 |
| D-1 | 修复与倒计时 | “明天，把新标签页留给一个问题” | 只修阻断发布的问题，锁定商店文案与素材 |
| D0 | 正式发布 | 主宣视频 + 商店链接 + 一句话定位 | 所有平台链接统一指向正式商店页 |
| D+1 | 降低使用门槛 | 30 秒安装与快捷键教程 | 观察安装问题、权限误解和新标签页冲突 |
| D+2 | 展示核心深度 | 一组“前因 → 命题 → 后续/分歧”卡片 | 引导讨论观点，不诱导刷五星评价 |
| D+3 | 场景化传播 | 上班开工、写作卡住、深夜浏览三个使用场景 | 比较哪个场景带来更多保存与自然转发 |
| D+4 | 社会证明 | 经本人同意整理 3 条内测反馈与对应改进 | 同时公开仍未解决的问题，避免只截好评 |
| D+5 | 讲创作故事 | 为什么取名“AGORA”，如何避免只做金句墙 | 强化独立产品的人格和设计判断 |
| D+6 | 首周复盘 | 已修复、待改进、下一批内容计划 | 发布可验证数据，不披露用户信息，不夸大规模 |

发布日不要六个平台机械复制同一段长文。统一主张和链接，但根据平台调整开头、长度与互动问题。

## 5. 30 秒首发视频脚本

| 时间 | 画面 | 旁白/字幕 |
|---:|---|---|
| 0–3 秒 | 连续打开三个普通新标签页，画面快速切换 | “你每天，会打开多少次新标签页？” |
| 3–7 秒 | 打开 AGORA，一则命题稳定出现 | “如果每一次，都能遇见一种想法呢？” |
| 7–13 秒 | 点击命题，详情层展开 | “它不只给一句金句：还有中英文译写与出处。” |
| 13–19 秒 | 切换思想脉络的前因、后续、相似/分歧观点 | “继续往下，是这句话从哪里来，又影响了谁。” |
| 19–23 秒 | 快速展示语言、双语开关和主题切换 | “三种语言、双语阅读，也保留你的阅读偏好。” |
| 23–26 秒 | Chrome 扩展详情页“无需权限”或产品内隐私说明 | “无账号、无追踪、无需权限。” |
| 26–30 秒 | 回到安静首屏，出现断环标记、产品名和商店入口 | “AGORA。一页，一种思想。” |

剪辑原则：命题正文保持静止可读，动效只用于背景结构或层级切换；不要用快速缩放让文字晃动。全片配字幕，音乐音量不得盖过旁白。

## 6. 内测清单

建议首轮 15–30 人，覆盖哲学读者、普通 Chrome 用户、设计/开发从业者和中英文读者。不要只找项目参与者。

### 安装与兼容

- [ ] macOS、Windows 至少各一台；条件允许时补充 ChromeOS/Linux；
- [ ] 当前稳定版 Chrome，以及一个较旧但仍受支持的版本；
- [ ] 1280×720、1440×900、1920×1080 和高 DPI 屏幕；
- [ ] 与常见新标签页扩展冲突时，提示清楚且不会循环报错；
- [ ] 商店安装、卸载、重新安装和更新后均能正常打开。

### 核心交互

- [ ] 点击命题能拉起详情，点击遮罩与 Esc 能关闭；
- [ ] 上一则、下一则、收藏、分享（不支持系统分享时复制到剪贴板）和主题切换可用；
- [ ] 菜单中的快捷键提示与实际快捷键完全一致；
- [ ] 焦点不会落到被弹层遮住的元素，键盘用户能完成主要操作；
- [ ] 中文长句优先按标点断行，不出现单字孤行；
- [ ] 动效可见但不带动正文，圆形元素不被拉伸成椭圆。

### 语言与内容

- [ ] 默认简体中文，“句子同时带英文”默认开启；
- [ ] 简体、繁体、英文切换后，命题、作者、书名、详情和设置项全部同步；
- [ ] 英文在上、中文在下时不额外堆叠“英文/中文”标签；
- [ ] 原典线索、双语译写、作者、书名和出处经至少两人交叉核验；
- [ ] 上下游关系能说明“继承、回应或反对”的具体原因，不只列名字；
- [ ] 相似观点与分歧观点不混为同一种关系；
- [ ] 对存在争议的归属或翻译明确标注，不把转述写成直接引文。

### 隐私与发布包

- [ ] `manifest.json` 没有 `permissions` 和 `host_permissions`；
- [ ] 断网仍可阅读核心内容；
- [ ] Network 面板没有第三方分析、字体、图片或内容请求；
- [ ] 本地只保存语言、主题、双语开关与收藏；
- [ ] 清除扩展数据后偏好恢复默认；
- [ ] 隐私说明的邮箱和生效日期已替换，占位符不进入正式商店页；
- [ ] ZIP 根目录直接包含 `manifest.json`，版本号高于已发布版本。

### 内测反馈模板

请内测者按以下格式反馈，便于复现：

```text
Chrome 版本：
操作系统与分辨率：
出现问题的页面/命题：
我做了什么：
我原本期待什么：
实际发生了什么：
能否稳定复现：
截图或录屏（如方便）：
内容纠错依据/来源（如涉及引文）：
```

发布阻断级问题：无法打开新标签页、数据明显错引、主要语言不可读、出现未披露的联网/权限、收藏或设置导致页面崩溃。其他优化进入首周迭代，不在发布前无限加功能。

## 7. 首周复盘框架

首周复盘只回答五个问题：

1. 商店展示到安装的转化是否符合预期？
2. 用户最常主动提到的是视觉、内容深度，还是使用便利？
3. 哪些命题最容易让人打开详情，依据来自公开互动与自愿反馈，而非隐蔽追踪？
4. 是否出现错引、错译或思想关系过度简化？修复时是否说明依据？
5. 下一版只解决哪三个最高价值问题？

对外复盘可以公布商店可见的聚合数字和自愿反馈，但不要推断个人浏览习惯，也不要为了好看的数据改变“零追踪”的产品承诺。
