# 设计来源与独立实现

AGORA 属于“新标签页 + 每日阅读”这一成熟产品类别。早期研究看过诗词、语录、冥想和知识卡片产品，目的是理解全屏单焦点阅读、低干扰控制和环境动效这些通用交互。

其中也研究过“几枝（Jizhi）”的公开产品与仓库，作为中文诗词新标签页的类别案例。检查时其公开仓库没有声明开源许可证，因此 AGORA 明确不使用它的源代码、数据、字体、传统色素材、插画或 p5.js 动画实现。

AGORA 不是几枝的移植或换皮。项目采用自己的信息与交互模型：

- 内容单位不是诗句，而是“命题译写 + 作品线索 + 解释 + 当下问题”；
- 详情层包含思想来处、同题异答、后续回响和观点对读；
- 三个视觉主题以思想轨迹、意识手稿和命题档案为概念；
- 环境动效由 CSS 的圆轨、节点、线框与明暗变化构成，不使用山形噪声、墨迹波浪或第三方动画素材；
- 实现采用 TypeScript、React、Next/vinext 和 Manifest V3，与所研究项目的组件与构建结构不同。

2026-07-21 的公开前审计还进行了跨仓库代码片段检查：在 JavaScript、TypeScript、TSX 和 JSX 文件中未检测到克隆片段。自动工具不能替代人的判断，因此以后任何贡献仍需说明来源，并遵守 [CONTRIBUTING.md](../CONTRIBUTING.md)。

研究一个类别并不意味着要隐藏影响。这个文档的作用，是把可接受的产品研究、独立实现和不可接受的复制边界公开写清楚。
